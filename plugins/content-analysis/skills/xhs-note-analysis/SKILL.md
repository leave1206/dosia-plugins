# 小红书笔记内容分析

## 概述

基于公司知识库中的内容框架体系，对小红书笔记进行结构化分析与评分。
覆盖标题公式识别、封面 7 维视觉分析、正文结构拆解、三感六度量化评分。

## Phase 1: 数据输入与解析

### 1.1 格式识别

根据用户提供的输入，自动识别数据格式：

| 输入类型 | 识别方式 | 解析方法 |
|---------|---------|---------|
| **本地 Excel** (.xlsx/.xls) | 文件路径以 `.xlsx` / `.xls` 结尾 | 用 Bash 执行 `python3 -c "import openpyxl; ..."` 读取 |
| **本地 CSV** (.csv) | 文件路径以 `.csv` 结尾 | 用 Bash 执行 `python3 -c "import csv; ..."` 读取 |
| **本地 JSON** (.json) | 文件路径以 `.json` 结尾 | 用 Read 工具直接读取 |
| **飞书电子表格** | URL 包含 `feishu.cn` 或 `larksuite.com` | 调用 `lark-sheets` skill 读取数据 |
| **用户粘贴的文本/表格** | 无文件路径，直接在消息中 | 直接从消息中解析 |

### 1.2 字段映射

从数据中提取以下核心字段（字段名可能因数据源不同而变化，需智能匹配）：

| 核心字段 | 可能的列名 | 必需 |
|---------|-----------|------|
| **笔记 ID** | 笔记ID, note_id, id | 否 |
| **标题** | 标题, title | 是 |
| **正文/摘要** | 内容摘要, 正文, content, body, summary | 是 |
| **笔记类型** | 笔记类型, type (图文/视频) | 是 |
| **封面图 URL** | 封面图, cover, cover_url, image_url | 推荐 |
| **作者** | 作者, author | 否 |
| **笔记链接** | 笔记链接, url, link | 否 |
| **话题标签** | 自动从正文中提取 `#xxx[话题]#` | 自动 |

### 1.3 Excel 读取模板

```python
import openpyxl
wb = openpyxl.load_workbook('FILE_PATH')
ws = wb[wb.sheetnames[0]]
headers = [cell.value for cell in next(ws.iter_rows(min_row=1, max_row=1))]
for row in ws.iter_rows(min_row=2, max_row=ws.max_row, values_only=True):
    if row[0] is not None:
        note = dict(zip(headers, row))
        print(f"--- 笔记 {note.get('序号', '')} ---")
        print(f"标题: {note.get('标题', '')}")
        print(f"摘要: {str(note.get('内容摘要', ''))[:200]}")
        print(f"类型: {note.get('笔记类型', '')}")
        print(f"封面: {note.get('封面图', '')}")
        print()
```

如果 `openpyxl` 未安装，先执行 `pip3 install openpyxl`。

## Phase 3: 封面图分析

### 3.1 图片获取流程（WebFetch → Read 两步法）

**封面图来自小红书 CDN，必须使用以下两步流程：**

```
Step 1: WebFetch 下载图片到本地
  ↓ WebFetch(url=封面URL, prompt="save")
  ↓ 忽略 WebFetch 的文本解析错误（它无法解析图片，这是预期行为）
  ↓ 从返回结果中提取本地保存路径：[Binary content ... saved to /path/to/file.jpg]

Step 2: Read 查看本地图片
  ↓ Read(file_path=本地保存路径)
  ↓ Claude 多模态能力直接理解图片内容
```

**关键规则：**
- WebFetch 返回的"无法处理"错误是正常的，不要中断流程
- 从 WebFetch 结果末尾提取 `saved to` 路径
- 每次 Read 一张图即可触发视觉理解
- 可以并行 WebFetch 多张图（5 张一批），然后并行 Read

### 3.2 封面 7 维评分

对每张封面执行 `multimodal-analysis.md` 中的 7 维视觉分析：

1. **构图与色彩** (0-100)
2. **信息层级** (0-100) — 0.3 秒首屏信息传达
3. **人物元素** (0-100)
4. **产品呈现** (0-100) — 产品/品牌是否可见且自然
5. **平台原生感** (0-100) — UGC 真实感，小红书审美匹配度
6. **文字设计** (0-100)
7. **封面类型** — 匹配 `patterns/cover-types.md` 中的 8 种类型

同时输出 **0.3 秒信息描述**：用户第一眼能获取什么信息。

### 3.3 批量图片处理策略

| 笔记数量 | 策略 |
|---------|------|
| ≤ 10 篇 | 全部封面逐张分析 |
| 11-30 篇 | 全部下载，全部分析 |
| > 30 篇 | 全部下载，全部分析（分批进行，每批 5 张） |

## Phase 4: 多维度分析与评分

### 4.1 内容分类

对每篇笔记进行以下分类（对照 `content-frameworks.md`）：

| 维度 | 分类选项 |
|------|---------|
| **内容类型** | 观点输出型 / 体验分享型 / 信息整合型 / 情绪共鸣型 |
| **内容目标** | 种草 / 口碑 / 教育 / 话题 / 转化 |
| **写作身份** | 行业观察者 / 深度用户 / 好奇尝鲜者 / 专业测评者 |
| **推广产品线** | 从正文/标签中识别具体产品线 |

### 4.2 标题评分

对照 `multimodal-analysis.md` 的标题 5 维分析 + `patterns/title-formulas.md` 的 7 种公式：

| 维度 | 评分 |
|------|------|
| **公式类型** | 匹配 7 种标题公式之一 |
| **钩子词** | 提取关键钩子词 |
| **信息密度** | 0-100 |
| **搜索友好度** | 0-100 |
| **情绪张力** | 0-100 |
| **标题综合分** | 三维均分 |

### 4.3 正文评分

对照 `multimodal-analysis.md` 的正文 5 维分析：

| 维度 | 评分 |
|------|------|
| **结构模型** | 匹配 4 种正文结构 |
| **段落节奏** | 0-100 |
| **干货密度** | 0-100 |
| **互动设计** | 识别互动引导元素 |
| **CTA 分析** | 行动号召位置和力度 |
| **正文综合分** | 加权均分 |

**自检清单执行**：对每篇笔记执行 `content-frameworks.md` 中的 7 项自检。
标记违规项（`fake_experience`, `ad_smell`, `emotion_match` 等）。

### 4.4 三感六度评分

严格按照 `three-senses-six-degrees.md` 的标准执行：

**六度评分（每维 0-100）：**
1. 受众精准度 (audience_precision)
2. 兴趣激发度 (interest_trigger)
3. 场景融入度 (scene_integration)
4. 情绪共鸣度 (emotion_resonance)
5. 需求满足度 (demand_satisfaction)
6. 自然呈现度 (natural_presentation) — 使用"删除法检验"

**三感计算：**
- 好奇感 = (受众精准度 + 兴趣激发度) / 2
- 认同感 = (场景融入度 + 情绪共鸣度) / 2
- 心动感 = (需求满足度 + 自然呈现度) / 2

**加权总分**：根据内容类型确定权重（激发好奇 / 激发认同 / 激发心动）。

**等级映射**：S(90-100) / A(80-89) / B(70-79) / C(60-69) / D(40-59) / E(0-39)

- 全局性优化建议

### 5.7 投放优先级建议

将笔记按综合评分分为 3-4 个梯队，给出具体投放建议。

## 特殊规则

### 视频笔记 vs 图文笔记

- **图文笔记**：封面 7 维分析 + 标题 5 维 + 正文 5 维 + 三感六度
- **视频笔记**：封面 7 维分析 + 标题 5 维 + 正文（摘要）分析 + 三感六度
  - 如果有完整口播文案：增加口播文案独立三感六度评分
  - 如果仅有摘要：基于摘要评分，标注"仅基于摘要，完整评分需口播文案"

### 正文不完整时的处理

如果数据源中正文被截断（常见于 API 导出的摘要字段）：
- 基于可用内容评分
- 在评分旁标注"(摘要)"
- 在总结中说明"正文评分基于摘要，完整评分需原始笔记全文"

### 封面图不可用时的处理

如果封面 URL 无法访问（CDN 过期、防盗链等）：
- 封面评分标记为"N/A"
- 不影响其他维度评分
- 在总结中统计不可用的封面数量

### 数量较多时的效率策略

| 总笔记数 | 策略 |
|---------|------|
| ≤ 15 篇 | 全维度完整分析，所有封面逐张拆解 |
| 16-50 篇 | 全维度评分表 + TOP5/BOTTOM5 详析 + 结构性发现 |
| > 50 篇 | 分批处理（每批 20 篇），最终汇总统计 + TOP10/BOTTOM10 详析 |
