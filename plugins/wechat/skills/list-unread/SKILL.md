# 微信未读消息列表技能

通过 wechat-cli 查当前所有未读消息（会话 + 具体消息）。

## 何时触发

- 用户说"帮我看看未读微信"、"有什么新消息"、"谁在找我"
- 用户回到工位/开会结束时："我这会儿有什么微信"
- 日常 standup 开始前的总览

## 使用方法

### 按会话聚合

```bash
wechat-cli sessions --has-unread true --format json
```

返回的 sessions 里 `unread_count > 0` 的行就是要展示的。

### 直接列未读消息

```bash
wechat-cli unread --format json
```

返回每条未读消息的：会话、发送者、时间、内容。

### 按重要性过滤

wechat-cli 不天生支持"重要"过滤，需要你基于返回内容判断：
- 群消息中 @ 当前用户的（content 含 `@你` 或特定格式）
- 直接好友发的（非群）
- 关键词"紧急"/"ASAP"/"please"

## 回复用户

默认行为：按 **重要性分组** 展示，而不是纯时间倒序。

```
🔴 @我 的群消息 (3 条)
- [项目群] 张三 @你 12:04: "这个需求的 XXX..."
- ...

👥 好友直接消息 (5 条)  
- 李四 12:30: "今晚吃啥"
- ...

📢 群消息 (23 条, 未显示)
> 说"展示群消息"查看详情
```

让用户快速判断哪些需要马上回。长消息截断到 50 字。

严禁自动"已读"（**不做任何 mark-as-read 操作**）— 未读状态是用户的
注意力信号，agent 不应清除。
