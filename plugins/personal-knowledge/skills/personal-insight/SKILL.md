# /personal-insight — 个人知识沉淀与成长 v2.0

> 非正式对话/文本 → 多领域知识萃取 → 个人知识库累积 → 成长追踪

---

## Step 0: 对话画像

读取文本后，先建立对话画像（输出到沉淀文档开头）：

### 0.1 场景类型识别

从以下类型中选择（可多选）：

| 类型 | 识别信号 | 示例 |
|------|---------|------|
| **商业头脑风暴** | 行业分析、竞争策略、商业模式讨论 | 茶室聊餐饮行业竞争 |
| **创业/产品讨论** | 产品定位、用户需求、MVP、融资 | 和朋友聊新产品方向 |
| **行业观察分享** | 趋势判断、案例分析、跨行业类比 | 聊韩国设计为什么超越日本 |
| **人生哲学探讨** | 价值观、做事方式、人生思考 | 聊"事以密成"的做事哲学 |
| **技术/AI 讨论** | AI 应用、技术趋势、工具使用 | 讨论 AI Agent 的窗口期 |
| **创意/审美讨论** | 设计、内容、艺术、文化现象 | 讨论反英雄叙事的社会学根源 |
| **读书/学习笔记** | 书籍内容、课程笔记、知识整理 | 读完《第一性原理》的思考 |
| **播客/访谈摘要** | 他人观点、行业人物访谈 | 听完某播客的关键收获 |

### 0.2 参与者识别

识别对话中的参与者（通过语言特征推断）：

| 角色 | 识别方式 | 在分析中的作用 |
|------|---------|--------------|
| **主要发言者** | 发言量最多、主导话题走向 | 核心观点来源、洞察主体 |
| **对话者** | 提问、回应、补充 | 触发深度思考的催化剂 |
| **被引用者** | 被提及的第三方（老师、专家等）| 权威观点来源 |

### 0.3 画像输出格式

```
场景类型: [类型1] + [类型2]（如有多个）
参与者: [姓名/代号] × N 人
核心话题: ① ... ② ... ③ ...
知识密度: [高/中/低]
提炼策略: [重点提炼方向说明]
```

## Step 2: 领域知识分类与提取

将对话中的知识点按 9 个领域分类提取。每个知识点包含：**原话引用 + 结构化解读 + 迁移应用**。

### 9 大知识领域

| 领域代码 | 领域名称 | 识别信号 |
|---------|---------|---------|
| **BIZ** | 商业策略与竞争 | 行业分析、竞争逻辑、商业模式、第一性原理 |
| **BRAND** | 品牌与体验 | 品牌定位、体验设计、品牌故事、差异化 |
| **CONSUMER** | 消费行为与心理 | 用户决策、消费趋势、心理洞察、人群变化 |
| **CONTENT** | 内容与创意 | 内容策略、创意方法、叙事结构、传播逻辑 |
| **DESIGN** | 设计与审美 | 视觉设计、空间设计、审美趋势、文化审美 |
| **AI_TECH** | AI 与科技趋势 | AI 应用、技术趋势、工具链、效率革命 |
| **PHILOSOPHY** | 人生哲学与价值观 | 做事方式、人生思考、价值判断、自我认知 |
| **CASES** | 商业案例库 | 具体公司/品牌/人物的案例分析 |
| **CULTURE** | 文化与社会观察 | 社会趋势、文化现象、国际比较、代际变化 |

### 每个知识点的输出格式

```markdown
### [领域代码] 领域名称

**知识点标题**

> "原话引用（保留口语感，适当修正语病）"

[结构化解读：将口语化的观点提炼为清晰的知识表述]

[迁移应用：这个知识点可以在什么场景下使用]
```

**提取原则**：
- 只提取有独立价值的知识点，不记录寒暄和过渡性对话
- 保留原话引用，保持思想的原始张力
- 结构化解读要补充口语中省略的逻辑链
- 迁移应用必须具体，不能泛泛而谈

## Step 3: 深度洞察萃取

从所有领域知识中，筛选出 3-5 个最有价值的深度洞察。

**筛选标准**（满足其一即可）：
- **反直觉**：与常规认知相反，提供新视角
- **可迁移**：适用于多个场景，不仅限于原始语境
- **第一性**：触及事物本质，而非表面现象
- **交叉洞察**：跨领域关联，产生 1+1>2 的新知

### 每个洞察的输出格式

```markdown
💡 **洞察标题（一句话总结）**
   来源语境: [在什么讨论中产生的]
   解释: [为什么这个洞察有价值，它揭示了什么]
   应用场景: [具体可以在哪些场景下使用]
```

---

## Step 4: 方法论框架提炼

将对话中隐含的分析框架、思维模型提炼为可复用的方法论。

**识别信号**：
- 有层级关系的分析（第一层、第二层...）
- 有步骤的流程（先...再...然后...）
- 有维度的评估（从X角度看、从Y角度看）
- 有对比的判断（A比B更重要，因为...）

### 每个方法论的输出格式

```markdown
### [方法论名称]

**适用场景:** [什么时候用]

**核心逻辑:**
1. [步骤/层级1]
2. [步骤/层级2]
3. ...

**原话摘录:** "[支撑这个方法论的关键原话]"

**可迁移到:** [其他适用领域]
```

## Step 5: 商业案例库

提取对话中提到的所有商业案例，结构化入库。

### 每个案例的输出格式

```markdown
#### [公司/品牌/人物名]
- **类型:** [成功案例/失败案例/正在验证/中性观察]
- **核心洞察:** [一句话总结这个案例的核心学习]
- **关键事实:** [数据、细节、时间线]（如有）
- **可借鉴点:** [对自己/公司有什么指导意义]
```

## Step 6: 公司启发

从个人对话中提取对公司/工作有启发意义的点。

### 输出格式

```markdown
**[启发标题]**
来源: "[相关原话引用]"
说明: [对公司业务/策略/方法论的具体启发]
状态: [强化已有 / 待验证 / 新发现]
```

**状态说明**：
- **强化已有**：与公司现有方法论/价值观一致，增强了信心
- **待验证**：有启发但需要在实际项目中验证
- **新发现**：全新的方向，值得探索

## Step 7: 个人成长

追踪对话中反映出的个人认知变化和成长信号。

### 三个维度

**自我认知**
- 对话中体现了哪些自我洞察？
- 有哪些思维模式被识别出来？
- 认知边界在哪里？

**当下思考**
- 当前正在思考什么核心问题？
- 有哪些未解决的困惑？
- 思考方向是否发生了变化？

**行动信号**
- 对话中产生了哪些行动意愿？
- 基于兴趣还是基于义务？
- 下一步可能的行动是什么？

## Step 8: 双视角点评

提供两个独立视角的点评，制造思想张力。

### Part A: 主要发言者的自我复盘视角

模拟主要发言者的内心独白：
- **本次对话收获最大的 3 个点** — 为什么它们最重要
- **说出的最有价值的判断** — 引用原话并解释价值
- **可以挖得更深但没挖的地方** — 遗漏了什么
- **对工作最直接的指导意义** — 落地行动

### Part B: 行业专家视角

以跨领域专家的身份独立点评：
- **行业基准对照** — 对话中的判断与行业数据/研究的吻合度
- **被忽视的视角** — 对话中没有触及但同样重要的角度
- **最有价值的洞察排序** — 按独创性和实用性排序
- **可能的盲点** — 观察偏差、过度简化、逻辑跳跃
- **建议深入研究的方向** — 哪些话题值得进一步探索

## 知识库累积机制

### 跨文档关联

每次生成新的知识沉淀文档时：
1. 用 Glob 扫描 `知识沉淀/` 目录下已有文档
2. 识别与已有知识点的关联（同领域的深化、相反的观点、新证据）
3. 在新文档中标注关联：`[关联: YYYY-MM-DD-xxx → 同主题深化]`

### 知识密度评估

| 密度 | 标准 | 说明 |
|------|------|------|
| **高** | ≥5 个独立知识点 + ≥2 个深度洞察 + ≥1 个方法论 | 值得反复阅读 |
| **中** | 3-4 个知识点 + 1 个深度洞察 | 有收获但不够密集 |
| **低** | <3 个知识点 | 主要是情感/社交价值，知识含量低 |
